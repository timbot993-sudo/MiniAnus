# Инструкция по настройке projects.json синхронизации

## Что произошло?

Теперь ваша система работает так:

1. **admin/admin.html** - кастомная админка для загрузки видео
2. Загруженные видео сохраняются в **projects.json**
3. **main.js** загружает видео из **projects.json**
4. При удалении видео - оно удаляется из **projects.json**

## Как настроить сохранение в GitHub?

Для автоматического сохранения projects.json в вашем репозитории нужно:

### 1. Создайте GitHub Personal Access Token

- Перейдите на https://github.com/settings/tokens
- Нажмите "Generate new token (classic)"
- Дайте права на `repo` (полный доступ к репозиториям)
- Скопируйте token

### 2. Добавьте переменные в Netlify

- Откройте Site settings на Netlify
- Перейдите в **Build & deploy** → **Environment**
- Нажмите **Edit variables**
- Добавьте:
  - `GITHUB_TOKEN` = ваш токен из шага 1
  - `GITHUB_OWNER` = ваше имя пользователя GitHub
  - `GITHUB_REPO` = название вашего репозитория

Пример:

```
GITHUB_TOKEN=github_pat_abc123...
GITHUB_OWNER=your-username
GITHUB_REPO=MiniAnus
```

### 3. Готово!

Теперь при загрузке видео через admin/admin.html:

- ✅ Видео сохранится в projects.json
- ✅ Все изменения синхронизируются с GitHub
- ✅ Все зрители видят актуальный список
- ✅ Удаленные видео исчезают из projects.json

## Альтернатива: Использовать встроенную Netlify CMS

Если вы не хотите мозолить с токенами, можете использовать встроенную админку:

- Перейдите на `/admin/` вместо `/admin/admin.html`
- Это встроенная Netlify CMS которая автоматически работает с GitHub

## Для локального развития (без Netlify)

Если вы развиваетесь локально, система будет использовать localStorage:

1. Загружайте видео через admin/admin.html
2. Данные сохраняются в localStorage браузера
3. При перезагрузке страницы - видео остаются
4. При развертывании на Netlify - автоматически синхронизируется в projects.json

## Если что-то не работает?

Откройте браузерную консоль (F12) и посмотрите ошибки при загрузке.
